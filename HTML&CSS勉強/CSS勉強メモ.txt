CSS勉強メモ

・全体の構造は「ルールセット」と呼ぶ。単にルールとも呼ばれる。構成要素は以下の通り。
　p {
    color: red;
}

    セレクター
    　→ pにあたる、ルールセットの先頭にあるHTML要素名。これはスタイルを設定する要素を定義する。

    宣言
    　→ color: red; にあたる、単一のルール。これは要素の「プロパティ」のうち、スタイル付けしたいものを指定する。

    プロパティ
    　→ HTML要素をスタイル付けするための方法。CSSでは、ルールの中で影響を与えたいプロパティを選択する。

    プロパティ値
    　→ redにあたる、プロパティの右側にはコロンの後にプロパティ値がある。

　セレクターを除き、それぞれのルールセットは{}で囲む必要がある。それぞれの宣言内では、コロンを使用してプロパティと値を分離する。
　それぞれのルールセット内でセミコロンを使用して、それぞれの宣言と次のルールを区切る必要がある。1つのルールセットで複数のプロパティ値を
　変更するには、以下のようにセミコロンで区切って書く。

    p {
        color: red;
        width: 500px;
        border: 1px solid black;
    }

・複数の要素の選択

　複数の要素を選択して、そのすべてに1つのルールセットを適用することもできる。複数のセレクターはカンマで区切る。例としては以下の通り。

    p,
    li,
    h1 {
        color: red;
    }

・様々な種類のセレクター

　セレクターには様々な種類がある。上記の例では「要素セレクター」を使用しており、特定の種類の要素を全て選択している。
　もっと特定の要素を選択することもできる。以下に一般的なセレクターの種類を記述する。

    要素セレクター
    　→ 指定された方の全てのHTML要素を選択。

    IDセレクター
    　→ 指定されたIDを持つページ上の要素。指定されたHTMLページでは、各id値は一意でなければならない。

    クラスセレクター
    　→ 指定されたクラスを持つページ上の要素。同じクラスの複数のインスタンスが1つのページに現れることがある。

    属性セレクター
    　→ 指定された属性を持つページ上の要素。

    疑似クラスセレクター
    　→　指定された要素が指定された状態にあるとき。(例えば、マウスポインターが上に乗っている状態。)

・HTML,CSS,JavaScriptといったwebの標準技術は全て「仕様書(または単に仕様)」と呼ばれる文書で定義されており、
　(W3C,WHATWG,ECMA,Khronosといった)標準化団体によって発行され、技術がどう働くよう期待されているかを定義している。

・ブラウザの対応状況は、MDNの各CSSプロパティページにある「ブラウザの互換性」と言う表に記載されている。

・CSS言語には、競合が発生した場合にどちらのセレクターが強いかを制御するためのルールがある。これらのルールは「カスケード」と
　「詳細度」と呼ばれている。例えば、以下のコードでは段落テキストは青くなる。

    p {
        color: red;
    }

    p {
        color: blue;
    }

　しかし、以下のコードでは段落テキストは赤くなる。

    .special {
        color: red;
    }

    p {
        color: blue;
    }

　これは、クラスセレクターと要素セレクターではクラスが優先されるため、赤が適用される。クラスは要素セレクターよりも詳細度が高いことに
　なっており、より具体的であると評価されるので、他の競合するスタイル宣言をキャンセルする。
　つまり、細かければ細かい指定ほどそれが優先される？

・font,background,padding,border,margin のようなプロパティは「一括指定プロパティ」と呼ばれている。
　これは一括指定プロパティが複数の値を1行で設定するため。

・警告：CSSの一括指定を使用する際に、省略した値は初期値に戻る。これは、CSS一括指定で省略された値は、
　「以前に設定された値を上書きしてしまう」可能性があることを意味する。
